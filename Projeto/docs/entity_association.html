Architecture note: The database is split into two levels — shared (vessels collection, visible to all users) and private (each user's own history, voyages, and saved vessels). This enables a collaborative vessel database where the first user to search a vessel by IMO populates the data for all future users.

Firestore Collection Schema

Firestore (root)
│
├── vessels/                        [SHARED — all users read/write]
│   └── {imo}/
│       ├── imo:              string    (PK — IMO number e.g. "9584188")
│       ├── name:             string    (vessel name)
│       ├── type:             string    (vessel type: Bulk Carrier, Tanker, ...)
│       ├── loa:              number    (length overall, metres)
│       ├── beam:             number    (breadth, metres)
│       ├── deadweight:       number    (DWT, tonnes)
│       ├── lightshipWeight:  number    (lightweight, tonnes)
│       ├── lightshipKG:      number    (lightship centre of gravity, metres)
│       ├── numberOfHolds:    number    (cargo holds count)
│       ├── flag:             string    (flag state ISO code)
│       ├── yearBuilt:        number    (year of construction)
│       ├── isEstimated:      boolean   (true if dimensions were auto-estimated)
│       ├── registeredBy:     string    (uid of first user who added this vessel)
│       └── lastUpdated:      timestamp
│
└── users/                          [PRIVATE — per authenticated user]
    └── {uid}/
        ├── profile/
        │   ├── displayName:  string
        │   ├── email:        string
        │   ├── plan:         string    ("free" | "pro")
        │   └── createdAt:   timestamp
        │
        ├── searchHistory/
        │   └── {id}/
        │       ├── imo:      string
        │       ├── name:     string
        │       └── searchedAt: timestamp
        │
        ├── savedVessels/           [user's personal fleet]
        │   └── {imo}/
        │       ├── imo:      string
        │       ├── name:     string
        │       └── savedAt:  timestamp
        │
        └── voyages/               [PRO feature — voyage history]
            └── {id}/
                ├── vesselImo:     string
                ├── vesselName:    string
                ├── departurePort: string
                ├── arrivalPort:   string
                ├── gmResult:      number
                ├── kgResult:      number
                ├── isStable:      boolean
                ├── totalCargo:    number
                ├── cargoData:     map       (hold name → weight)
                ├── marineConditions: map
                └── createdAt:     timestamp
